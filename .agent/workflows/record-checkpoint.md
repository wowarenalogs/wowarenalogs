# Workflow: Record Checkpoint

Whenever you and the user have an insightful discussion, reach a logical conclusion, finish a block of intense brainstorming, or reach a major milestone, you should automatically log it following these steps:

1. **Summarize the Conclusion:** Analyze what exactly was discussed and conceptually crystallized.
2. **Determine the Output Location:** 
   - If it is a feature backlog or set of future ideas, update `ANTIGRAVITY_IDEAS.md` or `AI_FEATURES.md`.
   - If it is an architectural decision or technical limitation, create or update a `.md` file in the root (like `ARCHITECTURE_DECISIONS.md`).
3. **Draft the Checkpoint:** Clearly write down the context, the exact decision or idea reached, and actionable blueprints for how it should be implemented by an agent later.
4. **Attribution:** Make sure to attribute who came up with the idea or what agent context it originated from (e.g., *Idea by Antigravity* or *Brainstormed by Claude Code*).
5. **Verify:** Use your file editing tools to append or create the markdown file with the finalized text. Let the user know the checkpoint is securely logged for the next agent or session.
