## Task 4: B38 — Promote major-CD spells from [OWNER CAST] to [OWNER CD]

**Files:**

- Modify: `packages/shared/src/components/CombatReport/CombatAIAnalysis/utils.ts` (around lines 1986–1993)
- Modify: `packages/shared/src/components/CombatReport/CombatAIAnalysis/__tests__/timeline.test.ts`
- Modify: `TRACKER.md`

**Root cause:** `HEALER_CAST_SPELL_ID_TO_NAME` includes major CDs like Avenging Crusader (216331, 60s CD). When `extractMajorCooldowns` fails to detect the spell (missing talent data), it isn't in `trackedCastsBySpellId`, so it falls through to `[OWNER CAST]` instead of `[OWNER CD]`. The fix: in the `[OWNER CAST]` section, when a spell has `cooldownSeconds >= 30` in `spellEffectData`, emit `[OWNER CD]` + resource snapshot instead of `[OWNER CAST]`.

## Task 5: B42 — Annotate dead players in [STATE] ticks

**Files:**

- Modify: `packages/shared/src/components/CombatReport/CombatAIAnalysis/utils.ts` (around lines 2186–2244)
- Modify: `packages/shared/src/components/CombatReport/CombatAIAnalysis/__tests__/timeline.test.ts`
- Modify: `TRACKER.md`

**Root cause:** `[STATE]` HP ticks call `getUnitHpAtTimestamp(unit, tsMs, …)` which returns `null` for dead players (no HP data after death). The `null` is filtered out, so dead enemies/allies silently disappear from the state. A dead enemy going from `e1:8` to absent looks like a data hole to Claude. Fix: build a death-time lookup before the tick loop and, for ticks after a player's death, emit `name:dead` rather than omitting the player.

- [ ] **Step 1: Write failing test**

In `timeline.test.ts`, find the `[HP] / [ENEMY HP] split` describe block (around line 2809) or add a new describe block and add:

```typescript
describe('buildMatchTimeline — B42: dead players remain visible in [STATE] ticks', () => {
  it('shows :dead for a friendly player in [STATE] ticks after their death', () => {
    const result = buildMatchTimeline(
      makeBaseParams({
        friends: [
          makeUnit('Feramonk', { spec: CombatUnitSpec.Paladin_Holy }),
          makeUnit('Simplesauce', { spec: CombatUnitSpec.DeathKnight_Unholy }),
        ],
        friendlyDeaths: [{ spec: 'Unholy Death Knight', name: 'Simplesauce', atSeconds: 20 }],
        matchStartMs: 0,
        matchEndMs: 60_000,
      }),
    );
    // At t=30 (past death), Simplesauce should show :dead not be omitted
    const linesAfterDeath = result.split('\n').filter((l) => {
      const m = l.match(/^(\d+):(\d+)\s+\[STATE\]/);
      if (!m) return false;
      const sec = parseInt(m[1]) * 60 + parseInt(m[2]);
      return sec >= 30;
    });
```
