## Notes

- Sprint 1 focus: Fix prompt quality before adding more data features
- Sprint 2 focus: Decision quality layer — timing, trade necessity, blame assignment
- Sprint 3 focus: Target selection, mirror benchmarking, avoidability
- The core shift needed: from "behavior analysis" to "decision analysis" — answer "was the trade correct" not "did you press the button"
- R1 player framing: "what did we trade, could we have avoided it, for what cost" — this is the question our system should answer
- See `AI_FEATURES.md` for full design philosophy, build order, and context structure spec
- AI test page at `/local/ai` (dev only) — validate all prompt changes before shipping
